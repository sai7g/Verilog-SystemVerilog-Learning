# Lexical Conventions in Verilog HDL

## Introduction

Lexical conventions define the basic rules for writing Verilog code. They specify how the Verilog compiler interprets characters, keywords, identifiers, numbers, operators, comments, and white spaces. Following these rules ensures that Verilog code is syntactically correct, readable, and easy to maintain.

---

# Components of Lexical Conventions

The lexical conventions in Verilog include:

- White Spaces
- Comments
- Identifiers
- Keywords
- Numbers
- Strings
- Operators
- Case Sensitivity

---

# 1. White Spaces

## Definition

White spaces improve the readability of Verilog code. They include spaces, tabs, and new lines. The Verilog compiler ignores extra white spaces except when they are needed to separate tokens.

### Types of White Spaces

- Space
- Tab
- New Line

### Example

```verilog
module and_gate(input a, input b, output y);
    assign y = a & b;
endmodule
```

---

# 2. Comments

## Definition

Comments are ignored by the compiler and are used to explain the purpose or functionality of the code.

### Single-Line Comment

```verilog
// This is a single-line comment
```

### Multi-Line Comment

```verilog
/*
This is
a multi-line comment
*/
```

### Uses

- Explain the functionality of the code.
- Improve code readability.
- Make maintenance easier.

---

# 3. Identifiers

## Definition

Identifiers are names assigned to modules, variables, signals, tasks, functions, and other user-defined objects.

### Rules for Identifiers

- Must begin with a letter (A–Z or a–z) or an underscore (_).
- May contain letters, digits, underscores (_), and dollar signs ($).
- Cannot begin with a digit.
- Cannot be a Verilog keyword.
- Are case-sensitive.

### Valid Identifiers

```text
clk
reset
data_in
_counter
temp$1
```

### Invalid Identifiers

```text
1data
module
always
input
```

---

# 4. Keywords

## Definition

Keywords are reserved words that have predefined meanings in Verilog. They cannot be used as identifiers.

### Common Verilog Keywords

```text
module
endmodule
wire
reg
assign
always
initial
if
else
case
begin
end
input
output
```

---

# 5. Numbers

## Definition

Verilog supports numeric constants in different number systems.

### General Format

```text
<size>'<base><value>
```

### Binary

```verilog
4'b1010
```

### Decimal

```verilog
8'd25
```

### Octal

```verilog
6'o37
```

### Hexadecimal

```verilog
16'h1A3F
```

### Examples

```verilog
8'b11001010
16'd255
8'hFF
4'o17
```

# 6. Strings

## Definition

A string is a sequence of characters enclosed within double quotation marks.

Strings are commonly used with system tasks such as `$display`.

### Example

```verilog
$display("Hello Verilog");
```

# 8. Case Sensitivity

## Definition

Verilog is a case-sensitive language.

This means that identifiers with different letter cases are treated as different names.

### Example

```verilog
reg data;
reg Data;
reg DATA;
```

The three identifiers above are considered different variables.
